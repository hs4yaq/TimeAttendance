🏫 PTC TimeAttendance Systemระบบลงเวลาปฏิบัติราชการ/ปฏิบัติหน้าที่ สำหรับบุคลากร วิทยาลัยเทคนิคภูเก็ต (Phuket Technical College) พัฒนาในรูปแบบ Web Application แบบ Single-Page และรองรับการทำงานเป็น PWA (Progressive Web App) เพื่อให้ผู้ใช้สามารถติดตั้งบนสมาร์ทโฟนได้เสมือนแอปพลิเคชันจริง✨ ฟีเจอร์เด่น (Key Features)📱 PWA Ready: รองรับการติดตั้งลงหน้าจอโฮม (Add to Home Screen) ทั้ง iOS (ผ่าน Safari) และ Android (ผ่าน Chrome)📍 Geofencing (GPS Tracking): ตรวจสอบพิกัดการลงเวลาแบบ Real-time โดยใช้ Leaflet.js ต้องอยู่ในรัศมีที่กำหนด (เช่น 200 เมตรจากวิทยาลัย) ถึงจะสามารถลงเวลาได้🔒 Device ID Binding: ระบบผูกรหัสพนักงานเข้ากับอุปกรณ์ (1 เครื่อง / 1 บัญชี) เพื่อป้องกันการฝากกันลงเวลา⏱️ Server Time Verification: ตรวจสอบเวลาผ่าน API ภายนอก (timeapi.io) ป้องกันการโกงเวลาโดยการปรับนาฬิกาในเครื่อง🛡️ Anti-Cheat & Channel Control: ดักจับและบล็อกการเข้าใช้งานผ่าน In-App Browser (เช่น LINE, Facebook) ที่มักทำให้พิกัด GPS คลาดเคลื่อน และบังคับให้เปิดในเบราว์เซอร์มาตรฐาน แอดมินสามารถเปิด-ปิดช่องทางได้แบบ Real-time👑 Admin Dashboard: แผงควบคุมสำหรับผู้บริหาร/ผู้ดูแลระบบดูรายงานการลงเวลาแบบ Real-timeจัดการบุคลากร และ ปลดล็อก Device IDตั้งค่าพิกัดวิทยาลัย, รัศมี, เวลาเข้า-ออกงาน ได้ผ่านหน้าเว็บส่งออกรายงานเป็น PDF และ Word (.doc)🛠️ เทคโนโลยีที่ใช้ (Tech Stack)Frontend: HTML5, Vanilla JavaScript (ES6 Modules)Styling: Tailwind CSS (via CDN)Icons: Lucide IconsMaps & GPS: Leaflet.js + OpenStreetMapBackend & Database: Firebase (Auth, Firestore)📂 โครงสร้างฐานข้อมูล (Firestore Structure)ระบบใช้ Google Cloud Firestore เป็นฐานข้อมูลหลัก โดยเก็บข้อมูลภายใต้ Collection artifacts/{appId}/public/data/:/users/{posId}: เก็บข้อมูลบุคลากร (ชื่อ, ตำแหน่ง, ฝ่าย, Device ID)/attendance/{docId}: เก็บประวัติการลงเวลา (พิกัด, เวลา, IP, สถานะ In/Out)/settings/config: เก็บการตั้งค่าระบบ (พิกัดวิทยาลัย, รัศมี, เวลาเปิด-ปิดระบบ, ช่องทางที่อนุญาต)/system_logs/{docId}: เกินประวัติการทำงานของ Admin (System Logs)🚀 การติดตั้งและใช้งาน (Installation)เนื่องจากโปรเจกต์นี้เป็น Client-side แทบทั้งหมด และพึ่งพา Firebase Backend คุณสามารถทำตามขั้นตอนดังนี้:1. การเตรียม Firebaseสร้างโปรเจกต์ใหม่ที่ Firebase Consoleเปิดใช้งาน Authentication (เปิดให้รองรับ Anonymous และ Google Sign-In)เปิดใช้งาน Firestore Databaseตั้งค่า Firestore Security Rules ให้เหมาะสม (ตัวอย่างแบบอนุญาตให้อ่าน/เขียนได้หาก Login):rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /artifacts/{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
